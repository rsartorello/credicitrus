# Diretrizes e Regras Gerais do Projeto

1. Não invente nenhuma informação.
2. Não apresente resumo do que foi realizado, apenas informe que foi concluído.
3. Não faça atualização em nenhum outro arquivo que não tenha ligação com a tarefa solicitada.
4. Se falarmos em responsividade, JAMAIS altere uma configuração para todas as resoluções, apenas para a solicitada.
5. Não atualize arquivos de tarefas ou histórico de atualizações (como `task.md`, `walkthrough.md`, `changelog.md`, etc.), o usuário controla o que foi atualizado.
6. Se possível, indicar qual foi o custo de token ao final de cada pedido.
